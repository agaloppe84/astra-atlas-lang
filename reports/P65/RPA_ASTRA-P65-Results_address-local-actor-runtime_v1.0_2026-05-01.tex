\section{But de ASTRA-P65}

ASTRA-P65 teste une architecture d'acteurs locaux d\'eterministes \`a budget.
Le but est de comprendre si le ratio ASTRA vient seulement de la g\'en\'eration
locale, ou du couple:

\[
  \text{adresse locale} + \text{acteur local \`a budget}
\]

Un acteur local est responsable d'un voisinage adress\'e \(N(A,r)\). Il peut
ex\'ecuter CRUD, audit, cache, \'eviction et compaction sans mat\'erialiser
l'espace virtuel global.

\section{Hypoth\`ese centrale: acteurs locaux adress\'es}

\begin{verbatim}
LocalActor(A, r) =
  neighborhood N(A, r)
  + cache local
  + index local
  + journal local
  + queue d'actions
  + audit local
  + budget bytes/temps/actions
  + regles de refus
\end{verbatim}

L'hypoth\`ese P65 est:

\begin{quote}
Un acteur local est une m\'emoire compt\'ee et limit\'ee. Il n'est utile que s'il
amortit plus de co\^ut qu'il n'en ajoute.
\end{quote}

Les acteurs P65 sont d\'eterministes. Ils ne sont pas des agents LLM et ne
portent aucune autonomie non d\'eterministe.

\section{Mod\`ele LocalActor}

Chaque \Code{LocalActor} expose au minimum:

\begin{itemize}
  \item identifiant d'acteur, adresse d'ancrage et rayon local;
  \item budget en octets et budget d'actions;
  \item cache, index, journal, queue, audit et compaction;
  \item octets d'\'etat, cache, index, journal, queue, audit et coordination;
  \item actions CRUD/audit/cache/evict/compact;
  \item conflits, lectures obsol\`etes et refus par budget.
\end{itemize}

Tout \'etat d'acteur est un co\^ut r\'eel. Toute coordination est un co\^ut
syst\`eme.

\section{Analyse de l'overhead acteur}

L'overhead de l'acteur local reste autour de 21\% du co\^ut total:

\begin{itemize}
  \item standard: \Code{actor_overhead_ratio = 0.212592};
  \item ambitious: \Code{actor_overhead_ratio = 0.214451}.
\end{itemize}

Ce niveau d'overhead est compatible avec un signal R\&D positif, mais il est
trop important pour pr\'etendre valider l'architecture finale. P65 mesure une
direction prometteuse, pas une limite scientifique.

\section{Analyse cache / journal / audit}

Le cache local est le facteur d'amortissement visible:

\begin{itemize}
  \item standard: \Code{cache_hit_rate = 0.473767};
  \item ambitious: \Code{cache_hit_rate = 0.536252}.
\end{itemize}

Les campagnes \Code{single_local_actor} ne produisent ni conflit ni lecture
obsol\`ete. Les strat\'egies plus fragment\'ees ajoutent de la coordination et
d\'egradent le ratio.

les seuils d'overhead, de cache, de journal, de rayon et de budget ne sont pas

\section{Ce que P65 \'etablit}

\begin{itemize}
  \item Les acteurs locaux peuvent am\'eliorer le ratio sur les fixtures P64.
  \item \Code{single_local_actor} est le meilleur candidat observ\'e.
  \item Les acteurs locaux am\'eliorent le ratio d'environ 33\%.
  \item Les strat\'egies trop sp\'ecialis\'ees ou trop fragment\'ees peuvent
        d\'egrader le syst\`eme.
  \item Le cache local est un levier concret d'amortissement.
\end{itemize}

\section{Ce que P65 ne prouve pas encore}

\begin{itemize}
  \item Pas de validation scientifique finale.
  \item Pas de dataset externe.
  \item Pas de campagne multi-machine.
  \item Pas de seuil calibr\'e pour l'overhead acteur.
  \item Pas de persistance longue dur\'ee d'un acteur r\'eel.
  \item Pas de golden de timing.
\end{itemize}

\section{Recommandation P65-2}

P65-2 doit calibrer:

\begin{itemize}
  \item overhead acteur acceptable;
  \item cache hit rate minimal;
  \item co\^ut journal/audit;
  \item rayon local;
  \item budget bytes/actions;
  \item seuils de conflits et stale reads.
\end{itemize}

La suite naturelle est de faire de \Code{single_local_actor} le candidat P66,
mais seulement apr\`es calibration plus stricte.

\section{Conclusion}

P65 valide l'int\'er\^et R\&D des acteurs locaux \`a budget. Le signal est clair:
\Code{single_local_actor} bat le baseline sans conflit ni stale read dans les
campagnes locales. Mais l'overhead autour de 21\% impose de rester en
\Decision{RECALIBRATE_P65_ACTOR_OVERHEAD}.
