\section{But de ASTRA-P68}

P68 transforme le signal P67 en d\'ecision architecturale robuste. Le sprint
ajoute un \Code{PromotionEvaluator} pairant un candidat standard et un candidat
ambitious, puis v\'erifie des gates stricts de gain, overhead, s\^uret\'e,
compatibilit\'e et comptabilisation CRUD/audit/compaction.

\section{Position apr\`es P67}

P67 avait trouv\'e une zone candidate forte:

\begin{itemize}
  \item standard: \Code{address_fiber_net_gain = 17.379955},
    \Code{avg_actor_overhead_ratio = 0.123446};
  \item ambitious: \Code{address_fiber_net_gain = 13.335472},
    \Code{avg_actor_overhead_ratio = 0.119345};
  \item conflits / stale reads / budget refusals: 0 / 0 / 0.
\end{itemize}

P67 restait toutefois en
\Decision{RECALIBRATE_P67_FIBER_OVERHEAD}, car la promotion exigeait une gate
pair\'ee standard + ambitious.

\section{Hypoth\`ese de promotion}

\Code{address_fiber_actor_managed_v1} devient candidat principal P69 si:

\begin{itemize}
  \item standard passe \Code{overhead < 0.15} et \Code{net_gain > 3.0};
  \item ambitious passe \Code{overhead < 0.18} et \Code{net_gain > 3.0};
  \item les candidats sont compatibles;
  \item conflits et stale reads restent nuls;
  \item les refus budgetaires restent sous 0.02;
  \item update, audit et compaction sont comptes.
\end{itemize}

\section{PromotionEvaluator}

Le \Code{PromotionEvaluator} produit:

\begin{itemize}
  \item \Code{standard_gate_status};
  \item \Code{ambitious_gate_status};
  \item \Code{pairing_status};
  \item \Code{promotion_score};
  \item \Code{passed_gates} et \Code{failed_gates};
  \item \Code{promotion_decision}.
\end{itemize}

\section{Ablations}

P68 ajoute des ablations causales locales:

\begin{itemize}
  \item cache off vs cache compact;
  \item journal lazy vs journal compact;
  \item audit sampled vs minimal;
  \item compaction off vs aggressive;
  \item metadata verbose vs standard;
  \item actor off vs actor-managed fiber;
  \item point fiber vs actor-managed fiber;
  \item projection medium vs shallow.
\end{itemize}

Ces contributions expliquent le protocole local. Elles ne sont pas presentees
comme lois scientifiques generales.

\section{Stress cibl\'es}

P68 teste douze stress: locality clustered/random/mixed/hotspot,
high-update, high-audit, metadata verbose, small budget, large radius, cache
churn, journal pressure et local/global conflict.

Les stress volontairement hors-classe peuvent retourner \Code{NO_GO} si le
refus est propre. La promotion exige qu'aucun stress raisonnable in-class ne
produise \Code{NO_GO}.

\section{Ce que P68 etablit}

\begin{itemize}
  \item Le candidat P67 passe les gates standard et ambitious.
  \item Le pairing est compatible.
  \item L'overhead reste sous seuil dans les deux modes.
  \item Les conflits, stale reads et refus budgetaires restent nuls.
  \item Les stress raisonnables ne produisent aucun \Code{NO_GO}.
  \item Une zone verte exploitable existe dans la phase map.
\end{itemize}

\section{Ce que P68 ne prouve pas encore}

\begin{itemize}
  \item Pas de validation scientifique finale.
  \item Pas de dataset externe.
  \item Pas de campagne multi-machine.
  \item Phase map compacte, non exhaustive.
  \item Stress scenarios protocolaires, non incidents production.
\end{itemize}

\section{Recommandation P69}

P69 doit implementer \Code{address_fiber_actor_managed_v1} comme candidat runtime
garde:

\begin{itemize}
  \item gates conflits/stale reads/budget refusals;
  \item comptage visible cache/journal/audit/metadata/compaction;
  \item replay sur plus de fixtures;
  \item preparation multi-machine avant revendication scientifique.
\end{itemize}

\section{Conclusion}

P68 clot le cycle P64--P68 par une promotion architecturale prudente mais
explicite. Le resultat principal n'est pas un benchmark isole; c'est un
evaluateur code qui transforme le signal P67 en decision reproductible.
