# ASTRA Patterns Catalog P76

## Promoted Or Stable Patterns

- Address-local generation: local generation dominates global materialization
  on realish workloads.
- Local actor: deterministic local actors improve address-local generation when
  overhead is counted.
- Address-fiber: data is represented as a fiber over an address.
- Representation contract: `.atlas` declares what procedural code and state are
  stored.
- Living procedural store: cold procedural state can be reopened into equivalent
  logical runtime state.
- Guard incompressible: random-like data is refused or raw/no-go, never credited
  as procedural success.
- Test stack hygiene: repo-first milestones audit Rust tests and justify
  changes.

## Candidate Patterns

- Mixed topology router: chooses topology by local fiber features.
- Routing oracle: evaluates route quality and wrong-route regret.
- Hierarchical tile: strong ratio/retrieval baseline on structured living data.

## Recalibrated Or Limited Patterns

- Cubical topology: useful in standard regular neighborhoods, but gluing cost
  can dominate under ambitious pressure.
- Single global topology: insufficient when update cost and retrieval fit pull
  in different directions.
- Declared cost model: must keep being checked against measured filesystem
  bytes.

## P77 Path

P77 should train or hand-calibrate deterministic router thresholds against P76
oracle regret while preserving living-memory 10 MiB campaigns.
